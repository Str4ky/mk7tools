Installation

In this part we will see how to install and use CHPACKer on other operating systems

warning

This section shows how to run CHPACKer on other operating systems, not how to MAKE a character on other operating systems

On Linux​

To install CHPACKer on Linux you will need these things

  • Flatpak
  • Bottles

So first, install Flatpak using the Linux distribution of your choice



Then install Bottles using this command on the terminal

flatpak install flathub com.usebottles.bottles

Then run it by going into your applications or by using this command

flatpak run com.usebottles.bottles

Then, create a new "Bottle" using bottles and click on it to go to it's settings

Click on "Browse Files..." to open your bottles's internal files and go to a place
like users/<user>/documents and place CHPACKer's and all of you character's files



info

Your bottle cannot access your Linux's internal storage, it can only access the storage of itself (the bottle)


Then on the main page, click on the cog icon to set CHPACKer's executable in your bottle and run it



And voilà, CHPACKer will now run and be ready to use :)


note

If your bottle says the app is not responding, hit "wait" until it shows the CHPACK saving prompt

On MacOS​

To install CHPACKer on Linux you will need this

  • Whisky

So first, install Whisky by clicking on this button



Then run it by going into your applications menu



Then, create a new "Bottle" using whisky and click on it to go to it's settings

Click on "Open C Drive" to open your bottles's internal files and go to a place
like users/<user>/documents and place CHPACKer's and all of you character's files



info

Your bottle cannot access your MacOS's internal storage, it can only access the storage of itself (the bottle)


Then on the main page, click on "Run..." and choose CHPACKer's
executable in order to run it through your bottle



And voilà, CHPACKer will now run and be ready to use :)


warning

Whisky is no longer maintained and things could be not working (text not showing but still working, CHPACK not properly generating, etc...) so be careful and aware that things might not 100% work as intended
